# Job Stories — Week 6 Needs-Finding

## Story 1: Keyboard Submission Clarity  
**Situation**: When I’m adding a task using only the keyboard  
**Motivation**: I want clear feedback that the Enter key will submit the form  
**Outcome**: So I don’t waste time guessing or switching to a mouse  
**Evidence**: Participant A (notes 2025-11-08, time 14:31)  
**Inclusion Risk**: Affects keyboard-only users, motor impairment users  


## Story 2: Focusable Delete Buttons  
**Situation**: When I need to delete a task using Tab navigation  
**Motivation**: I want delete buttons to appear in the focus order  
**Outcome**: So I can access all features without a mouse  
**Evidence**: Participant A (notes 2025-11-08, time 14:33)  
**Inclusion Risk**: Critical for keyboard users, screen reader users  


## Story 3: Screen Reader Confirmations  
**Situation**: When I complete an action (add/delete task)  
**Motivation**: I want the screen reader to announce success/failure  
**Outcome**: So I know the action worked without visual checks  
**Evidence**: Participant B (notes 2025-11-10, time 10:16)  
**Inclusion Risk**: Essential for screen reader users  


## Story 4: Inline Editing  
**Situation**: When I need to correct a task description  
**Motivation**: I want to edit tasks directly in the list  
**Outcome**: So I don’t waste time navigating to new pages  
**Evidence**: Participant B (notes 2025-11-10, time 10:18)  
**Inclusion Risk**: Benefits users with cognitive load issues, motor impairments  


## Story 5: Clear Error Messages  
**Situation**: When I make a mistake in a form (e.g., blank task)  
**Motivation**: I want specific error messages (e.g., "Please enter a task description")  
**Outcome**: So I can fix the issue quickly without confusion  
**Evidence**: Participant B (notes 2025-11-10, time 10:20)  
**Inclusion Risk**: Affects all users, particularly those with cognitive disabilities  


## Story 6: Scannable Task List  
**Situation**: When I’m reviewing a long list of tasks  
**Motivation**: I want clear visual hierarchy (e.g., spacing, labels)  
**Outcome**: So I can find important tasks quickly  
**Evidence**: Participant A (notes 2025-11-08, time 14:35)  
**Inclusion Risk**: Benefits low-vision users, users with ADHD
